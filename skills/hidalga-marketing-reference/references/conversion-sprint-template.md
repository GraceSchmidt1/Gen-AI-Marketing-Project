# Monthly Conversion Experiment Sprint Template

*Source: Q1_Monthly_Conversion_Experiment_Sprint_Template.docx — Hidalga Technologies*

---

## Purpose
Run structured content experiments for one focus ICP segment to improve click-to-key-action performance by at least **10% over baseline** during a single monthly sprint.

---

## Sprint Overview Fields

| Field | Description |
|-------|-------------|
| Sprint Month | Month / Year |
| Focus ICP Segment | e.g., COO / CFO / RCM leaders at small to mid-sized community oncology clinics |
| Primary Pain Point | e.g., denial-related revenue loss, delayed treatment starts, staff rework, lack of PA visibility |
| Primary Key Action | Select ONE: Demo booking / PA calculator visit / Newsletter signup / Blog click-through / Contact form submission |
| Baseline Period | Dates |
| Experiment Window | Dates |
| Success Threshold | At least 10% lift in click-to-key-action rate vs. baseline |
| Owner | Name |
| Reviewer/Approver | Joshua or other approver |

---

## Baseline Snapshot

Capture before the experiment begins:
- Baseline content sample dates
- Number of comparable posts reviewed
- Baseline click-through rate
- Baseline click-to-key-action conversion rate
- Baseline total key actions
- Top-performing content pillar during baseline
- Top-performing format during baseline (static graphic / video / carousel / text-led post / event recap)
- Top-performing CTA during baseline
- Notes on what appears to be working and what appears weak

---

## Experiment Hypothesis Structure

**Template:**
> If we change [hook / CTA / format / proof point / landing page angle] for [ICP segment], we expect click-to-key-action rate to improve because [reason tied to audience pain point].

Document the hypothesis clearly before beginning the experiment window.

---

## During the Sprint
- Execute content according to the hypothesis
- Tag all experimental posts in Loomly and Planner with the relevant ICP and content pillar
- Note any mid-sprint observations

---

## Sprint Outcome Review
At close of experiment window, document:
- Actual click-to-key-action rate achieved
- Lift vs. baseline (positive or negative)
- Whether the 10% threshold was met
- What worked and why (specific post examples)
- What didn't work and why
- Recommendation for next sprint
